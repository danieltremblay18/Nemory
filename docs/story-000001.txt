## PROJECT NAME

Nemory

---

## PROJECT VISION

Build a modern Progressive Web App (PWA) called **Nemory**.

The goal is to create the simplest possible application for keeping a personal history of maintenance, inspections, repairs, services, and recurring reminders for personal assets.

This application is NOT an inventory management system.

It is NOT an enterprise asset management platform.

It is NOT an ERP.

It is simply a personal maintenance journal.

At any time, the user should be able to answer five questions:

* What was done?
* When was it done?
* On which asset?
* What are the details?
* When should it be done again?

If a feature doesn't directly support this vision, don't build it.

## ASSET

An Asset is simply something I own or want to remember maintenance for.

Examples:

* House
* Cottage
* King Generator
* Polaris Sportsman 570
* Honda Foreman
* UV Water System
* Septic Tank
* Roof

Nothing more.

No inventory.

No serial number.

No purchase date.

No purchase price.

No depreciation.

No accounting.

No complex asset management.

---

## ACTIVITY

An Activity represents something that happened to an asset.

Examples:

* Oil Change
* Annual Inspection
* UV Lamp Replacement
* Roof Repair
* Filter Replacement
* Tire Rotation
* Septic Tank Pumping

Each Activity contains:

* Asset
* Title
* Description
* Activity Date
* Optional Reminder Interval
* Automatically Calculated Next Reminder Date

Future versions may include:

* Photos
* PDF invoices
* Attachments
